# Proposal: 客户端直连后端迁移规范

## 问题背景

当前项目采用 React Router v7 的 SSR 架构，登录认证通过服务端 action 处理。前期已完成登录页面的客户端直连迁移（client-side-login-localstorage），但其他页面仍使用服务端代理模式。

随着前端交互需求增加，需要将更多页面改为客户端直连后端，以提升用户体验和减少服务端负担。

## 目标

建立统一的迁移规范，指导后续页面从服务端请求迁移到客户端直连后端。

## 方案概述

### 架构设计

```
客户端直连架构:
┌─────────────┐     CORS      ┌─────────────┐
│   浏览器    │ ───────────▶  │   Django    │
│  (Vite)     │               │   Backend   │
└─────────────┘               └─────────────┘
      │
      ▼ localStorage
┌─────────────┐
│  Token 存储 │
└─────────────┘
      │
      ▼
┌─────────────┐     Cookie    ┌─────────────┐
│  SSR Session │ ───────────▶  │   服务端    │
│  (_session)  │               │  (可选)     │
└─────────────┘               └─────────────┘
```

### 关键设计决策

1. **双 Token 存储**
   - `localStorage`: 存储 JWT access/refresh token，用于客户端 API 请求
   - `Cookie (_session)`: 存储服务端 session，用于 SSR 和服务端渲染

2. **HTTP 客户端**
   - 使用 `axios` 作为 HTTP 库
   - 使用 `app/utils/http/client.ts` 中的 `clientHttp` 和 `clientAuth`

3. **Session 同步**
   - 客户端登录后需调用 `/auth/set-session` 设置服务端 session
   - 登出时使用服务端路由 `/auth/logout` 清除 session

4. **路由类型选择**
   - **客户端路由**: 纯展示页面，无需 SSR 数据
   - **服务端路由**: 需要 SEO、需要服务端预渲染的页面

## 实施计划

1. 创建迁移检查清单
2. 定义组件改造标准
3. 编写示例代码模板
4. 验证方案可行性

## 成功标准

- 后续迁移工作有明确的规范可循
- 新增页面可快速判断是否需要迁移
- 代码风格保持一致
